# 超越物理内存：机制

物理内存不够用时，OS 把暂时不用的页放到磁盘上（**交换空间**），用到时再换回来。让进程以为自己有比实际物理内存大得多的地址空间。

## 交换空间

磁盘上专门划一块区域存放被换出的页，叫**交换空间**（swap space）。OS 以页为单位在交换空间和物理内存之间来回搬数据。

注意：程序代码页不需要写回交换空间，因为可以直接从程序的二进制文件重新加载。

## 存在位

页表项里加一个**存在位**（present bit）：

- 1 = 页在物理内存中，正常翻译
- 0 = 页不在内存里（被换出到磁盘或者还没加载进来）

访问一个存在位为 0 的页 → 触发**页错误**（page fault）。

## 页错误处理

页错误由 OS 的**页错误处理程序**处理（不管是硬件还是软件管理的 TLB，页错误都交给 OS）：

1. 从 PTE 里读出该页在磁盘上的地址
2. 找一个空闲物理页帧（如果没有，先用页替换算法踢掉一页）
3. 发起磁盘 I/O，把页读进物理内存
4. 更新 PTE：设置 PFN，存在位改为 1
5. 重试触发页错误的那条指令

磁盘 I/O 期间，触发页错误的进程被**阻塞**，OS 调度其他进程运行。这就是多道程序充分利用 CPU 的方式。

## 硬件控制流

```text
VPN = 从虚拟地址中提取
查找 TLB:
  命中 → 检查权限 → 访问物理内存
  未命中 → 查页表:
    PTE 无效 → SEGFAULT
    PTE 有效但不存在（present=0）→ PAGE_FAULT（OS 接管）
    PTE 有效且存在 → 装入 TLB → 重试
```

## 水位线机制

OS 不会等内存完全用光才开始换出。设置两个阈值：

- **低水位线**（LW）：空闲页低于此值时，后台的**交换守护进程**（swap daemon / page daemon）开始换出页面
- **高水位线**（HW）：换出到空闲页达到此值时停止

批量换出（聚集写入）能利用磁盘顺序写的高吞吐，比每次换一页高效得多。
