# 进程

进程就是运行中的程序。程序本身是静态的，存在磁盘上；操作系统把它跑起来，就成了进程。

## 机器状态

进程能读写的所有东西，构成它的机器状态（machine state）：

- **地址空间**：进程能访问的内存
- **寄存器**：程序计数器（PC）记录下一条指令地址；栈指针（SP）和帧指针（FP）管理函数调用栈
- **I/O 信息**：进程打开的文件列表（文件描述符）

## 进程 API

操作系统提供的进程操作接口：

- **创建**（create）：生成新进程
- **销毁**（destroy）：强制杀死进程
- **等待**（wait）：等某个进程结束
- **状态查询**（status）：获取进程信息

## 进程创建过程

1. 把程序代码和静态数据加载到内存（现代系统用懒加载，按需加页）
2. 分配栈空间，初始化 `argc`/`argv`
3. 分配堆空间
4. 做其他初始化（比如打开 fd 0/1/2 对应 stdin/stdout/stderr）
5. 跳到 `main()` 入口，OS 把 CPU 控制权交给进程

## 进程状态

三种基本状态：

- **运行**（Running）：正在 CPU 上执行指令
- **就绪**（Ready）：可以运行但 OS 还没调度到它
- **阻塞**（Blocked）：在等某个事件（如 I/O 完成），不能运行

状态转换：运行 ↔ 就绪（由调度决定），运行 → 阻塞（发起 I/O），阻塞 → 就绪（I/O 完成）。

还有 **僵尸态**（zombie）：进程已退出但父进程还没调用 `wait()` 回收。

## 进程控制块

OS 用 **PCB**（Process Control Block）记录每个进程的信息。以 xv6 为例，`proc` 结构体包含：进程状态、页表指针、内核栈、打开的文件列表、当前目录等。`context` 结构体保存寄存器（用于上下文切换）。
