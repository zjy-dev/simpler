# 分页

分页是现代 OS 管理内存的核心机制。把物理内存切成固定大小的块（**页帧**，通常 4KB），进程的地址空间也按同样大小切成**页**（page）。任意一页可以放在任意一个页帧里。

## 页表

OS 为每个进程维护一个**页表**（page table），记录虚拟页到物理页帧的映射。

虚拟地址 = VPN（虚拟页号）+ offset（页内偏移）

翻译过程：从虚拟地址取出 VPN → 查页表得到 PFN → 拼上 offset → 物理地址。

以 64 字节地址空间、16 字节页为例：虚拟地址 21 = 二进制 `010101`，高 2 位 `01` 是 VPN=1，低 4 位 `0101` 是偏移=5。查页表 VPN 1 → PFN 7，物理地址 = 7×16+5 = 117。

## 页表项（PTE）

页表的每一项包含多个标志位：

| 标志位 | 含义 |
|--------|------|
| 有效位（valid） | 该页是否被进程使用。未使用的虚拟页标记无效，访问会触发 trap |
| 保护位（protection） | 读/写/执行权限 |
| 存在位（present） | 该页是否在物理内存中（还是被换出到磁盘了） |
| 脏位（dirty） | 页被写过，换出时需要回写磁盘 |
| 参考位（reference/accessed） | 页被访问过，用于页替换算法 |

## 页表太大的问题

32 位地址空间 + 4KB 页 = $2^{20}$ 个页表项。每项 4 字节的话，单个页表就 4MB。系统里跑 100 个进程就是 400MB 纯页表开销。加上每次内存访问都要多一次查页表的访问，性能也受影响。

这两个问题分别由 TLB（下一章）和多级页表来解决。
