# 磁盘驱动器

## 物理结构

磁盘由一个或多个**盘片**（platter）组成，每面有磁性涂层。盘片绕**主轴**高速旋转（7200~15000 RPM）。**磁头**在**磁臂**末端，悬浮在盘面上方读写数据。

每个盘面被划分为同心圆的**磁道**（track），每条磁道分为若干**扇区**（sector，通常 512B）。

## I/O 时间

$$T_{I/O} = T_{seek} + T_{rotation} + T_{transfer}$$

- **寻道时间**（$T_{seek}$）：磁头移动到目标磁道。包含加速、滑行、减速、稳定，典型 0.5~2ms（相邻磁道）到数毫秒（全程）
- **旋转延迟**（$T_{rotation}$）：等目标扇区转到磁头下方。平均等半圈。7200 RPM 的盘一圈约 8.3ms，平均旋转延迟约 4.15ms
- **传输时间**（$T_{transfer}$）：读写数据，取决于数据量和盘面线速度，通常最短

## 顺序 vs 随机

这是面试高频考点。以 Cheetah 15K 磁盘为例：

- 顺序读：~125 MB/s
- 随机读：~0.66 MB/s

差距约 **200 倍**。原因：随机读每次都要寻道+旋转，顺序读几乎不用。

## 磁盘调度

OS（或磁盘控制器）对 I/O 请求重排序以减少磁头移动：

**SSTF**（最短寻道时间优先）：选离当前磁头最近的请求。贪心，可能饿死远端请求。

**SCAN / C-SCAN**（电梯算法）：磁头从一端扫到另一端，沿途处理请求。C-SCAN 到达一端后直接跳回起点重新扫，对所有位置的请求更公平。

**SPTF**（最短定位时间优先）：同时考虑寻道和旋转延迟，选总定位时间最短的请求。效果最好，但 OS 通常不知道磁盘内部的精确布局，所以 SPTF 一般在磁盘固件里实现。

## 其他优化

- **I/O 合并**：相邻扇区的多个请求合并成一个大请求
- **磁道偏斜**（track skew）：相邻磁道的 0 号扇区错开几个位置，避免跨磁道读时因旋转错过
- **磁盘缓存**（track buffer）：读整条磁道到缓存。写策略分两种：**write back**（写入缓存就返回，快但断电丢数据）和 **write through**（写到盘面才返回，安全但慢）
