# 数据库

## 数据库结构

Redis 服务器默认创建 **16** 个数据库（`dbnum` 配置项），用 `SELECT` 命令切换。每个 `redisDb` 结构有两个关键字典：

- **键空间**（`dict *dict`）：存所有键值对，键是字符串对象，值是任意类型对象
- **过期字典**（`dict *expires`）：键指向键空间中的键对象，值是毫秒级 UNIX 时间戳

`EXPIRE`/`PEXPIRE`/`EXPIREAT`/`PEXPIREAT` 四个命令底层都转换为 `PEXPIREAT` 执行。

## 过期键删除策略

三种方案：

| 策略 | 优点 | 缺点 |
|------|------|------|
| 定时删除 | 内存友好，到期即删 | CPU 不友好，创建大量定时器 |
| 惰性删除 | CPU 友好，只在访问时检查 | 内存可能泄漏，大量过期键堆积 |
| 定期删除 | 折中 | 频率和时长需要调参 |

**Redis 采用惰性删除 + 定期删除组合：**

- **惰性删除：** 所有读写命令执行前调用 `expireIfNeeded()`，如果键过期就删除
- **定期删除：** `serverCron` 周期性调用 `activeExpireCycle()`，每次随机检查若干数据库，每个库随机取 20 个键检查

## 过期键与持久化、复制

- **RDB 生成**：跳过过期键
- **AOF 重写**：跳过过期键
- **主从复制**：主服务器删除过期键后向从服务器发 `DEL` 命令；从服务器不会自主删除过期键，必须等主服务器通知
