# 事件

Redis 是事件驱动的，靠文件事件和时间事件运转。

## 文件事件

Redis 用 I/O 多路复用监听多个套接字，采用 Reactor 模式：

```text
套接字 → I/O 多路复用程序 → 队列 → 文件事件分派器 → 事件处理器
```

多路复用的底层实现可以是 `epoll`、`kqueue`、`evport`、`select`，编译时自动选最高效的。多个套接字就绪时逐个处理，同一套接字优先处理读事件再处理写事件。

### 常用事件处理器

- **连接应答处理器**：监听套接字的 `AE_READABLE` 事件，接受新连接
- **命令请求处理器**：客户端套接字的 `AE_READABLE` 事件，读取并执行命令
- **命令回复处理器**：客户端套接字的 `AE_WRITABLE` 事件，返回结果给客户端

一次完整的客户端交互：
1. 客户端连接 → 触发连接应答处理器 → 创建客户端套接字，绑定命令请求处理器
2. 客户端发命令 → 触发命令请求处理器 → 执行命令，绑定命令回复处理器
3. 客户端可读 → 触发命令回复处理器 → 写回结果，解除绑定

## 时间事件

时间事件存放在无序链表中（数量少，遍历不影响性能），分两类：

- **定时事件**：指定时间执行一次
- **周期事件**：每隔一段时间执行（处理器返回下次间隔毫秒数）

`serverCron` 是 Redis 最核心的周期时间事件（默认 100ms 一次），负责：
- 更新统计信息（内存占用、连接数等）
- 清理过期键
- 检查自动保存条件
- 进行 AOF/RDB 相关后台操作
- 集群模式下的定期同步

## 事件调度

事件循环中，先计算最近的时间事件截止时间作为阻塞上限，然后阻塞等待文件事件。文件事件到达就处理，之后检查并执行到期的时间事件。文件事件和时间事件都是同步、有序、原子执行，不会互相抢占。
