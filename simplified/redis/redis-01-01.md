# 简单动态字符串（SDS）

Redis 不用 C 原生字符串，而是自己造了 SDS（Simple Dynamic String）。数据库里所有字符串值的底层都是 SDS，AOF 缓冲区和客户端输入缓冲区也用它。

## 结构定义

```c
struct sdshdr {
    int len;    // 已使用字节数（字符串长度）
    int free;   // 未使用字节数
    char buf[]; // 字节数组，末尾自动补 '\0'
};
```

末尾的 `'\0'` 不计入 `len`，由 SDS 函数自动维护。好处是可以直接复用 C 标准库的部分函数（如 `printf("%s", s->buf)`）。

## SDS vs C 字符串

| 特性 | C 字符串 | SDS |
|------|---------|-----|
| 获取长度 | O(N) 遍历 | O(1) 读 `len` |
| 缓冲区溢出 | 拼接前忘分配就溢出 | API 自动检查并扩容 |
| 内存重分配 | 改 N 次必分配 N 次 | 改 N 次最多分配 N 次 |
| 数据类型 | 只能存文本 | 二进制安全，图片音频都行 |
| 兼容 C 函数 | 全部 | 部分（因为保留了 `'\0'`） |

## 空间预分配

扩容时不只分配刚好够的空间，还会多分配一块：

- `len < 1MB`：额外分配与 `len` 等量的 `free` 空间（`buf` 实际大小 = `len + len + 1`）
- `len >= 1MB`：额外分配固定 1MB

效果：连续增长 N 次字符串，内存重分配次数从必定 N 次降到最多 N 次。

## 惰性空间释放

缩短字符串时不立即回收多余内存，而是记到 `free` 里留着以后用。需要时可以手动调 API 真正释放。

## 二进制安全

C 字符串遇到 `'\0'` 就认为结束了，SDS 用 `len` 判断长度，中间可以有任意字节。所以 SDS 能存任何二进制数据。
