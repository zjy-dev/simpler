# 连接的原理

## 连接的本质

连接查询就是把各个表中的记录取出来依次匹配组合，加入结果集返回。两张表直接连接产生的结果集称为**笛卡尔积**——t1 有 3 条记录、t2 有 3 条记录，笛卡尔积就是 3×3=9 行。

```sql
SELECT * FROM t1, t2;  -- 产生笛卡尔积
```

实际使用中必须加过滤条件来减少结果集，过滤条件分两类：

- **单表条件**：只涉及一张表，如 `t1.m1 > 1`
- **两表条件**：涉及两张表，如 `t1.m1 = t2.m2`

连接查询的执行步骤：

1. 确定**驱动表**，用单表条件对驱动表执行查询，得到结果集
2. 对结果集中的每一条记录，带入两表条件去**被驱动表**中查找匹配记录

驱动表只访问一次，被驱动表的访问次数取决于驱动表结果集的记录条数。

## 连接类型

### 内连接

驱动表中的记录在被驱动表中找不到匹配记录时，**不会**加入结果集。

MySQL 中以下写法等价：

```sql
SELECT * FROM t1, t2 WHERE t1.m1 = t2.m2;
SELECT * FROM t1 JOIN t2 ON t1.m1 = t2.m2;
SELECT * FROM t1 INNER JOIN t2 ON t1.m1 = t2.m2;
SELECT * FROM t1 CROSS JOIN t2 ON t1.m1 = t2.m2;
```

内连接中 `ON` 子句和 `WHERE` 子句等价，驱动表和被驱动表可以互换，不影响结果。

### 左/右外连接

驱动表中的记录即使在被驱动表中没有匹配记录，**仍然**加入结果集，被驱动表对应字段用 `NULL` 填充。

```sql
-- 左外连接：左表为驱动表
SELECT * FROM t1 LEFT JOIN t2 ON t1.m1 = t2.m2;

-- 右外连接：右表为驱动表
SELECT * FROM t1 RIGHT JOIN t2 ON t1.m1 = t2.m2;
```

以 t1 = {1,2,3}、t2 = {2,3,4} 为例，三种连接结果对比：

| 连接类型 | 结果行数 | 特殊行 |
|---------|---------|--------|
| INNER JOIN | 2 | 无 |
| LEFT JOIN | 3 | t1 中 m1=1 的记录保留，t2 列为 NULL |
| RIGHT JOIN | 3 | t2 中 m2=4 的记录保留，t1 列为 NULL |

外连接的驱动表和被驱动表不能互换，左连接驱动表固定为左表，右连接固定为右表。外连接必须使用 `ON` 子句指定连接条件。

### ON 子句与 WHERE 子句的区别

两者在外连接中语义不同：

- **WHERE 子句**：不论内连接还是外连接，不符合条件的记录一律从结果集中剔除
- **ON 子句**：对于外连接，驱动表记录在被驱动表中找不到满足 ON 条件的匹配时，该记录仍保留在结果集中，被驱动表列用 NULL 填充

对于**内连接**，ON 和 WHERE 完全等价。

实践建议：单表过滤条件放 WHERE，两表连接条件放 ON。

## 连接的执行原理

### 嵌套循环连接（Nested-Loop Join）

最基本的连接算法。伪代码：

```
for each row in t1 {           -- 遍历驱动表结果集
    for each row in t2 {       -- 遍历被驱动表
        if row satisfies join conditions, send to client
    }
}
```

三表连接则再嵌套一层循环。驱动表只访问一次，被驱动表访问次数 = 驱动表结果集行数。

![嵌套循环连接示意图](images/11-04.png)

这种算法简单但效率低——如果被驱动表无索引，每次都要全表扫描。

### 使用索引加速连接

被驱动表的每次查询本质上就是一次单表查询，可以利用索引加速。例如：

```sql
SELECT * FROM t1, t2 WHERE t1.m1 > 1 AND t1.m1 = t2.m2 AND t2.n2 < 'd';
```

对 t2 的查询相当于 `SELECT * FROM t2 WHERE t2.m2 = 2 AND t2.n2 < 'd'`，可利用的索引策略：

| 索引列 | 访问方法 | 说明 |
|-------|---------|------|
| m2 | ref | 等值查找，回表后判断 n2 条件 |
| m2（主键/唯一索引） | eq_ref | 等值查找代价为常数级 |
| n2 | range | 范围查找，回表后判断 m2 条件 |

`eq_ref` 是连接查询特有的访问方法：对被驱动表使用主键或唯一二级索引列做等值查找。

两个索引都可用时，优化器选择代价更低的那个。索引代价高于全表扫描时不会使用索引。

如果查询列表和过滤条件涉及的列恰好都属于某个索引，还可以走**索引覆盖扫描**（index 访问方法），避免回表。因此不要用 `SELECT *`，只查需要的列。

### 基于块的嵌套循环连接（Block Nested-Loop Join）

普通嵌套循环连接中，驱动表结果集有多少条记录，被驱动表就要从磁盘加载多少次，I/O 代价极高。

优化思路：把多条驱动表记录一起加载到内存，一次性和被驱动表记录做批量匹配，减少被驱动表的磁盘读取次数。

MySQL 引入了 **join buffer**：

1. 将若干条驱动表记录装入 join buffer
2. 扫描被驱动表，每条记录和 join buffer 中的所有驱动表记录做匹配
3. 匹配完成后清空 buffer，装入下一批驱动表记录，重复步骤 2

![Block Nested-Loop Join 示意图](images/11-06.png)

理想情况下 join buffer 能容纳驱动表全部结果集，被驱动表只需扫描一次。

#### join_buffer_size

- 通过系统变量 `join_buffer_size` 配置，默认 **256KB**（262144 字节）
- 最小值 128 字节
- join buffer 中只存放**查询列表中的列和过滤条件中的列**，不是整行数据

因此 `SELECT *` 不仅浪费网络带宽，还会让 join buffer 中能存放的记录数减少，间接增加被驱动表的扫描次数。

优化优先级：为被驱动表建立合适的索引 > 调大 join_buffer_size。

## 驱动表的选择

- **内连接**：优化器自由选择驱动表，通常选结果集较小的表作为驱动表（减少被驱动表的访问次数）
- **左外连接**：驱动表固定为左表
- **右外连接**：驱动表固定为右表

核心原则：**小表驱动大表**。这里的"小表"指的是经过过滤条件筛选后结果集行数更少的表，而非原始数据量更少的表。
